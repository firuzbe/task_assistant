# 📌 Task Assistant Bot (Telegram + Email + Voice)

Telegram-бот для постановки задач сотрудникам с поддержкой:

- ✅ постановки задач текстом и голосом
- 📧 отправки задач на Email
- 📲 отправки задач в Telegram
- 📢 массовой рассылки задач всем сотрудникам
- 🧠 автоматического распознавания исполнителя и дедлайна (GigaChat)
- 🎤 распознавания речи (Whisper)
- ✅ кнопки "Выполнено" для закрытия задачи
- 🗂 хранения задач в SQLite базе данных


## 🚀 Возможности

### 👨‍💼 Руководитель / Админ может:
- ставить задачу конкретному сотруднику: `/newtask`
- делать массовую рассылку задачи всем сотрудникам: `/broadcast`
- смотреть список активных задач: `/tasks`
- смотреть историю задач: `/history`
- отменять задачи: `/cancel ID`
- управлять сотрудниками: добавлять / удалять / привязывать Telegram ID


### 👷 Сотрудник может:
- войти по email (пароль = email)
- просматривать свои задачи: `/mytasks`
- отмечать задачу выполненной через кнопку **"✅ Выполнено"**


## 🏗 Структура проекта

```

task_assistant/
│
├── bot.py               # Основной Telegram бот
├── database.py          # Работа с SQLite базой
├── config.py            # Конфигурация из .env
├── email_sender.py      # Отправка email уведомлений
├── deadline_parser.py   # Парсер дедлайнов (завтра, до пятницы и т.п.)
├── gigachat.py          # Анализ текста задачи через GigaChat
├── asr.py               # Распознавание голоса Whisper (faster-whisper)
│
├── tasks.db             # SQLite база (создаётся автоматически)
├── logs/
│   └── bot.log          # Логи работы бота
│
└── .env                 # Переменные окружения (не хранить в git!)

````



## ⚙️ Установка

### 1) Клонируем проект

```bash
git clone https://github.com/your_repo/task_assistant.git
cd task_assistant
````



### 2) Устанавливаем зависимости

```bash
pip install -r requirements.txt
```

⚠️ Для Whisper рекомендуется установить `ffmpeg`.

**Windows:**

* скачать ffmpeg и добавить в PATH

**Linux:**

```bash
sudo apt install ffmpeg
```


## 🔑 Настройка `.env`

Создай файл `.env` в корне проекта:

```env
# Telegram
TELEGRAM_TOKEN=YOUR_TELEGRAM_BOT_TOKEN
ADMIN_TELEGRAM_ID=123456789

# Пароль для руководителя/админа
BOT_PASSWORD=admin123

# GigaChat API
GIGACHAT_AUTH_KEY=...
GIGACHAT_AUTH_URL=...
GIGACHAT_API_URL=...

# Email SMTP
SMTP_SERVER=smtp.gmail.com
SMTP_PORT=587
EMAIL_LOGIN=your_email@gmail.com
EMAIL_PASSWORD=your_password

# Limits
MAX_ACTIVE_TASKS=25

# DB and Logs
DB_FILE=tasks.db
LOG_FILE=logs/bot.log
```

## 👥 Авторизация

### 👨‍💼 Админ / Руководитель

Авторизация выполняется через общий пароль:

```
/start
admin123
```

Пароль берётся из `.env` (`BOT_PASSWORD`).


### 👷 Сотрудник

Пароль сотрудника = его email.

Пример:

```
/start
ivanov@company.ru
```

Если такой email есть в базе `employees`, бот автоматически:

* авторизует сотрудника
* привяжет его Telegram ID к профилю


## 📌 Команды

### Общие команды

| Команда   | Описание             |
| --------- | -------------------- |
| `/start`  | вход                 |
| `/logout` | выход                |
| `/help`   | справка              |
| `/id`     | показать telegram_id |


### Задачи

| Команда      | Описание                                |
| ------------ | --------------------------------------- |
| `/newtask`   | создать задачу сотруднику (текст/голос) |
| `/tasks`     | список всех активных задач              |
| `/history`   | история задач                           |
| `/cancel ID` | отменить задачу                         |
| `/mytasks`   | задачи сотрудника                       |


### Админ

| Команда                            | Описание                         |
| ---------------------------------- | -------------------------------- |
| `/employees`                       | список сотрудников               |
| `/add_employee ФИО email`          | добавить сотрудника              |
| `/del_employee email`              | удалить сотрудника               |
| `/bind_employee email telegram_id` | привязать Telegram ID сотрудника |
| `/broadcast`                       | массовая рассылка задачи всем    |


## 🎯 Постановка задачи сотруднику (`/newtask`)

### Пример текстом:

```
/newtask Иванов, подготовить отчет по продажам до пятницы
```

### Пример голосом:

1. отправить команду:

```
/newtask
```

2. отправить голосовое сообщение

Бот:

* распознает речь
* определит исполнителя и дедлайн
* покажет подтверждение
* отправит задачу на email + telegram исполнителю


## 📢 Массовая рассылка (`/broadcast`)

### Пример текстом:

```
/broadcast Подготовить общий отчет по продажам до пятницы
```

### Пример голосом:

1. отправить команду:

```
/broadcast
```

2. отправить голосовое сообщение

Бот:

* создаст задачу
* назначит всем сотрудникам
* отправит всем email
* отправит всем telegram сообщение (если telegram_id привязан)


## ✅ Выполнение задачи

Когда сотрудник получает задачу в Telegram, появляется кнопка:

**✅ Выполнено**

После нажатия:

* задача переводится в статус `done`
* руководителю приходит уведомление в Telegram


## 📂 Статусы задач

| Статус      | Описание        |
| ----------- | --------------- |
| `active`    | активная задача |
| `done`      | выполнена       |
| `cancelled` | отменена        |


## 🗃 База данных (SQLite)

Используется файл `tasks.db`.

Основные таблицы:

* `employees` — сотрудники
* `tasks` — задачи
* `task_assignees` — связь задача ↔ сотрудник
* `bot_users` — пользователи Telegram (авторизация, роль)


## 📝 Логи

Логи сохраняются в:

```
logs/bot.log
```
